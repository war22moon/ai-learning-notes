# AI Learning Notes

面向售前解决方案经理、产品经理、项目经理的 AI 行业阶段性学习手册。

这个仓库用于沉淀我的 AI 行业学习成果。它不是每日推送的原始归档，也不是未经筛选的信息流，而是把阶段性学习内容整理成可复习、可引用、可分享的结构化手册。

## 学习目标

- 用 100 天建立 AI 行业的核心概念地图。
- 理解大模型服务从模型资产、推理引擎到 API 平台的完整链路。
- 掌握模型服务、AI 应用构建、RAG、Agent、AI Coding、推理基础设施、商业模式、SLA 与政企交付等关键主题。
- 将技术概念转化为客户能听懂、合同能落地、项目能交付的表达方式。

## 更新节奏与维护原则

这个仓库只保存阶段性总结或精选手册，保持高信噪比和可复盘性。

- 每日学习推送和当天追问保存在本机 OneDrive 的 `perplexity/100天AI学习计划/YYYY/MM/YYYY-MM-DD/`，不自动同步到 GitHub。
- GitHub 不维护每日归档，也不再自动维护每周学习周报。
- 只有在我主动触发阶段性总结时，才把整理后的精选内容同步到 GitHub。
- 阶段性总结优先放在 `stage-summaries/`，每个阶段一个独立目录。
- 每个阶段目录应尽量包含阅读说明、正式手册、必要的对话归档和课程参考资料。

## 第 1 周核心问题

- DeepSeek、Qwen、Claude 到底是模型、模型资产，还是模型服务？
- 开发框架、模型资产、推理引擎、模型 API 之间是什么关系？
- CUDA、PyTorch、vLLM、SGLang、TensorRT-LLM 分别对应哪些工作？
- 国产芯片上的 CANN、MindIE、vLLM-Ascend 应该如何理解？
- MCP 和普通 API、插件、RAG、Agent 有什么区别？
- AI Gateway 和国内常见的 Token 中转站是不是一类东西？
- Token 计费模式下应该关注哪些成本、性能和折算指标？
- 流式与非流式返回对用户体验、计费和 SLA 有什么影响？
- P90、P95、P99 在 AI API 服务质量中如何使用？
- 面向政企客户，AI API 服务的 SLA 应该如何写进合同？

## 适合读者

这个仓库适合具备云服务、售前解决方案、项目交付或产品管理背景，但并非 AI 算法工程师的人阅读。目标是把 AI 行业中的关键概念转化为可沟通、可报价、可交付、可复盘的业务知识。
