# AI Learning Notes

面向售前解决方案经理、产品经理、项目经理的 AI 行业学习笔记与周报。

这个仓库用于沉淀我的 AI 行业学习过程，重点不是算法论文推导，而是从云服务、模型服务、API 平台、基础设施、商业模式、SLA 与政企交付视角，逐步建立可用于业务沟通和项目判断的知识框架。

## 学习目标

- 用 100 天建立 AI 行业的核心概念地图。
- 理解大模型服务从模型资产、推理引擎到 API 平台的完整链路。
- 掌握模型聚合平台、AI Gateway、Token 计费、SLA、成本治理等售前和交付高频议题。
- 将技术概念转化为客户能听懂、合同能落地、项目能交付的表达方式。

## 更新节奏与维护原则

这个仓库刻意保持“高信噪比、结构化、可复盘”，不是一个原始信息流。因此更新节奏遵循以下原则：

- **每日 Perplexity / 应用内推送**只用于学习输入和短期捕捉，属于个人学习过程，不直接进入仓库。
- **GitHub 不会每天自动同步**，也不再维护每日推送归档。
- **GitHub 只在以下情况更新**：
  - 完成每周的精选/筛选式总结后；或
  - 用户主动要求阶段性总结时。
- **优先产出**放在 `weekly-reports/`，后续可选地补充 `concept-cards/`（概念卡片）或精选主题指南。
- 目标是让仓库始终保持**高信号、结构化、可审阅**，而不是变成未经筛选的原始推送堆叠。

## Week 01 核心问题

- DeepSeek、Qwen、Claude 到底是模型、模型资产，还是模型服务？
- 开发框架、模型资产、推理引擎、模型 API 之间是什么关系？
- CUDA、PyTorch、vLLM、SGLang、TensorRT-LLM 分别对应哪些工作？
- 国产芯片上的 CANN、MindIE、vLLM-Ascend 应该如何理解？
- MCP 和普通 API、插件、RAG、Agent 有什么区别？
- AI Gateway 和国内常见的 Token 中转站是不是一类东西？
- Token 计费模式下应该关注哪些成本、性能和折算指标？
- 流式与非流式返回对用户体验、计费和 SLA 有什么影响？
- P90、P95、P99 在 AI API 服务质量中如何使用？
- 面向政企客户，AI API 服务的 SLA 应该如何写进合同？
